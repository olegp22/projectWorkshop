## 👤 Аналитик

### 1. Сбор и формализация требований
- Описать пользовательские роли:
  - Организатор
  - Проверяющий (эксперт / студент / жюри)
- Зафиксировать user flow для каждого режима:
  - Экспертная проверка
  - Peer-to-peer оценка
  - Конкурсное голосование
- Определить обязательные и опциональные функции

### 2. Проработка бизнес-логики
- Описать процесс:
  - загрузки работ
  - постановки в очередь
  - распределения работ
  - отправки на проверку
  - возврата результатов
- Описать правила распределения:
  - исключение самопроверки
  - ограничение количества проверок
  - защита от дублирования проверок
- Продумать критерии оценки:
  - шкалы
  - веса (если есть)
  - агрегирование результатов

### 3. Проектирование данных
- Определить структуру БД / Google Sheets:
  - пользователи
  - работы
  - проверки
  - критерии
- Описать связи между сущностями
- Подготовить схему (ER-диаграмму)

### 4. API и интеграции
- Сформулировать требования к API:
  - Telegram-бот
  - Web-интерфейс
- Определить интеграции:
  - Google Sheets API
  - Speech-to-Text (Whisper / Google)

## 💻 Разработчик

### 1. Базовая инфраструктура
- Настроить репозиторий (Git)
- Настроить структуру проекта
- Выбрать стек:
  - Telegram-бот (aiogram / python-telegram-bot)
  - Backend (FastAPI)
  - БД или Google Sheets

### 2. Авторизация и пользователи
- Авторизация через Telegram ID
- Реализация инвайт-ссылок
- Разделение ролей

### 3. Очередь работ
- Реализовать очередь:
  - добавление работ
  - получение следующей работы
- Логика блокировки:
  - одна работа — один проверяющий (по умолчанию)
- Настройки распределения

### 4. Логика проверки
- Отправка работы пользователю
- Приём ответа:
  - текст
  - голос
- Интеграция с Speech-to-Text:
  - транскрибация голосовых

### 5. Система оценивания
- Поддержка критериев
- Ввод оценок
- Расчёт агрегированных результатов

### 6. Хранение данных
- Интеграция с Google Sheets API или БД
- Сохранение:
  - работ
  - пользователей
  - проверок
- Обновление статусов

### 7. Реализация режимов (минимум 2)
- Экспертная проверка
- Peer-to-peer:
  - автоматическое распределение
- Конкурс:
  - голосование
  - рейтинг

### 8. Web-интерфейс (опционально)
- FastAPI backend
- Простая админка:
  - загрузка работ
  - просмотр результатов

### 9. Тестирование
- Проверка полного цикла:
  - загрузка → распределение → проверка → результат
- Обработка ошибок

### 10. Демо и финализация
- Подготовка сценария демонстрации
- Заполнение тестовыми данными
- Проверка всех ролей
